Cuentas bancarias

Cada mes defines el saldo inicial y el saldo real de corte por cuenta. Con eso el sistema calcula el saldo esperado según movimientos y detecta el descuadre.

Mes Cuenta Saldo inicial Saldo real

Cómo usar esta pantalla

  • Saldo inicial: el saldo bancario con el que arrancó el mes.
  • Saldo real: el saldo que muestra Bancolombia el día del corte.
  • Si el saldo real no coincide con el esperado según movimientos, el dashboard lo marca en rojo.
Para un cierre diario, puedes usar el mismo mes y actualizar el saldo real todas las tardes. Así ves si el descuadre contra movimientos es cero.

Provisiones y obligaciones internas

Usa esta pantalla para partidas que sí afectan la caja real disponible pero que no necesariamente salen del banco el mismo día: ICA, compras por volumen, nómina pendiente, reservas técnicas.

Enfoque recomendado

  • IVA y retenciones salen de liquidaciones y pagos reales.
  • ICA y compras por volumen pueden provisionarse aquí para no engañarte con una caja “aparente”.
  • Si después lo conviertes en app completa con base de datos, esta tabla será una entidad aparte y muy útil para alertas.

Ruta recomendada

  1. Cargar saldos por cuenta del mes.
  2. Pegar o importar movimientos diarios.
  3. Subir liquidaciones resumidas por apartamento.
  4. Registrar provisiones que no quieres perder de radar.
  5. Mirar el dashboard. Si el descuadre no es 0, ir a reportes y buscar por rubro/cuenta.
Este MVP sirve para validar la lógica y operar ya. Un agente autónomo real necesitaría un backend con usuarios, base de datos, reglas, y eventualmente integración bancaria.

Mantenimiento de la base local

Los datos se guardan en localStorage del navegador. Para trabajo real, exporta el JSON con frecuencia o migra esto a una base de datos.
